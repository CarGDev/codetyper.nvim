# Codetyper.nvim — LLM Context File

> Version: 1.1.0

> License: MIT
> Neovim: >= 0.8

## What is Codetyper.nvim?

A Neovim plugin that acts as an AI-powered coding partner. It uses LLM APIs
(GitHub Copilot, Ollama) to help write, refactor, explain, and organize code
while staying in the editor. Features an agent system that can create files,
move functions, and manage imports across a project.

## Key Concepts

### Model Tiers
Models are classified by capability:
- **agent** (Claude, GPT-4o, o3): tool calls, FILE: operations, TOOL: calls, multi-step reasoning
- **chat** (GPT-4o-mini, Copilot default): strict format, explicit line boundaries
- **basic** (codellama, small Ollama): fill-in-the-middle, minimal prompt

Tier auto-detected from Copilot /models API (tool_calls capability) or hardcoded in model_caps.lua.

### Agent Operations
When using agent-tier models, the LLM can output:
- `FILE:CREATE path` — create a new file
- `FILE:MODIFY path` + SEARCH/REPLACE — modify existing files
- `FILE:DELETE path` — delete files
- `TOOL:TERMINAL command` — run shell commands
- `TOOL:MCP server/tool {args}` — call MCP tools

The agent loop supports multi-turn: LLM calls tools, results fed back, LLM continues (max 5 iterations).

### Brain/Memory System
The brain is a graph-based knowledge store in `.codetyper/brain/`:
- Learns coding style on file save (indent, module patterns, error handling, guards)
- Learns project architecture from tree.log (which directories hold what code)
- Learns from successful injections (what conventions were used)
- Targeted queries: file conventions, intent patterns, scope context (not a context dump)
- Nodes: patterns, corrections, conventions, metrics
- Edges: semantic, file-based, temporal relationships

### Prompt Pipeline
1. User prompts via `<leader>ctt` (selection or cursor) or `/@ @/` tags
2. Intent detected (fix, add, refactor, explain, etc.)
3. Scope resolved via Tree-sitter (function, method, class, file)
4. Context gathered (brain, architecture, indexed files, attached files)
5. Prompt built using model-tier-specific builder (agent/chat/basic)
6. Sent to LLM via Copilot or Ollama
7. Response parsed for agent operations (FILE:, TOOL:) or plain code
8. Code injected at correct position using extmark-tracked ranges

### Tag System (/@ @/)
Inline tags in code (must be inside comments for valid syntax):
```lua
-- /@ add error handling to this function @/
```
- Default: manual processing via `:CoderProcess`
- Toggle auto-processing: `:CoderAutotrigger`
- Multiple tags processed sequentially bottom-to-top

## Commands

| Command | Description |
|---------|-------------|
| `:Coder transform-selection` | Open prompt window |
| `:Coder cost` | Toggle cost tracking window |
| `:Coder terminal` | Toggle terminal panel |
| `:Coder queue` | Toggle prompt queue window |
| `:Coder autotrigger` | Toggle /@ @/ auto-processing |
| `:Coder process` | Manually process /@ @/ tags |
| `:Coder model [name]` | Switch Copilot model |
| `:CoderTransformSelection` | Open prompt window |
| `:CoderCost` | Toggle cost window |
| `:CoderAutotrigger` | Toggle autotrigger |
| `:CoderProcess` | Process /@ @/ tags |

## Keymaps

| Key | Mode | Description |
|-----|------|-------------|
| `<leader>ctt` | Normal/Visual | Open prompt window |
| `<leader>ter` | Normal | Toggle terminal |
| `@` | Insert (prompt window) | Open file picker to attach files |

## Configuration

```lua
require("codetyper").setup({
  llm = {
    provider = "copilot",  -- or "ollama"
    copilot = { model = "claude-sonnet-4" },
    ollama = { host = "http://localhost:11434", model = "llama3" },
  },
  scheduler = {
    enabled = true,
    apply_delay_ms = 500,
  },
  brain = { enabled = true },
  indexer = { enabled = false },
})
```

## .codetyper/ Directory

Per-project data stored in `.codetyper/` (auto-gitignored):
- `brain/` — Knowledge graph (nodes, edges, indices, deltas)
- `agents/` — Agent persona definitions (.agent.md)
- `tree.log` — Project file structure
- `cost_history.json` — Token usage tracking
- `models_cache.json` — Cached Copilot model capabilities
- `settings.json` — Project-specific settings
